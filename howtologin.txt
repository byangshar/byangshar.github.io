You don’t actually “log in” to Git itself — you authenticate when you push or pull. The cleanest way on Windows is: install Git for Windows, then run git config --global user.name "YourName" and git config --global user.email "you@example.com". After that, go to GitHub → Settings → Developer settings → Personal access tokens → Fine-grained tokens and create a token. The first time you do git push, Git will ask for a username (use your GitHub username) and a password (paste the token). Windows Credential Manager then saves it, so future pushes won’t ask again.
